# Screenshots Checklist (UI changes)

Include screenshots/recordings when any user-facing UI behavior, layout, accessibility, or copy changes.

## Capture at least

- The primary “happy path” screen/state.
- Any error/empty/loading states affected by the change.
- Responsive checks (at least one narrow + one wide viewport) if layout is involved.

## Notes to include

- What the screenshot demonstrates (one sentence).
- Any known limitations or follow-ups.
